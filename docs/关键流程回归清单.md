# 关键流程回归清单（2026-06-13 修复后验证）

> 目的：本轮触及「金额联动、权限隔离、并发、导入/同步」等核心，合入 main 前建议在测试环境逐条验证。

## 1. 收付款 ↔ 合同 paid_amount 联动（手动入口，回归）
1. 新建一份销售合同 A（金额 10000，状态 active）。
2. 新建收款：业务类 `business`、关联合同 A、金额 3000、已确认 `confirmed`。
   - ✅ 合同 A `paid_amount` = 3000，进度 30%。
3. 编辑该收款金额 3000 → 5000。
   - ✅ 合同 A `paid_amount` = 5000。
4. 删除该收款。
   - ✅ 合同 A `paid_amount` = 0。
5. 新建一条 `pending` 业务类收款关联 A，金额 2000 → 再「确认」。
   - ✅ 确认前 `paid_amount` 不变；确认后 +2000。

## 2. 企微同步付款联动（本次修复 A，重点）
> 无企微环境可用「手动造一条等效 payment」间接验证，或在有企微时走审批。
1. 准备采购合同 B（金额 8000，active，记下其 `sp_no` 或在同步映射中关联）。
2. 触发企微付款审批同步（business + confirmed，关联合同 B，金额 5000）。
   - ✅ 合同 B `paid_amount` = 5000（**修复前为 0**）。
3. 触发一条费用类（fee + confirmed，带成本类别）同步付款。
   - ✅ `cost_records` 新增一条对应记录（**修复前不写**），Dashboard 成本饼图计入。

## 3. 历史导入付款联动（本次修复 A）
1. 用「数据导入」上传一批收付款：含 1 条 business+confirmed 关联已存在合同 C，1 条 fee+confirmed 带类别。
   - ✅ 合同 C `paid_amount` 增加对应金额；fee 行在 `cost_records` 生成记录。
2. 合同导入：状态填 `pending` 的行。
   - ✅ 导入成功，落库状态为 `draft`（**修复前整批报错失败**）。

## 4. 详情接口越权读（本次修复，重点安全）

1. 找一条「别人创建」的收付款 id，直接请求 `GET /api/v1/payments/:id`。
   - ✅ 返回 404「收付款记录不存在」（**修复前会返回他人记录**）。
2. 同样验证 `GET /inventory/:id`、`/loans/:id`、`/expenses/:id`、`/projects/:id` 别人创建的记录。
   - ✅ 均 404。
3. agent 访问「自己创建」的上述详情。
   - ✅ 正常返回。
4. admin / process 访问任意详情。
   - ✅ 正常返回（不受隔离）。

## 5. 账户转账并发与余额（修复 #4）
