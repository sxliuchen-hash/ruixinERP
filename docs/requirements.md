## 项目简介

为10人专利买卖公司开发内部财务管理系统：
- 与企业微信审批打通，审批通过后数据自动同步
- 实时展示公司现金流、利润、成本等核心指标
- 管理囤积专利的库存、维持成本、库龄
- 作为 `patent-notice-system` 的子项目，复用技术栈和用户体系

## 核心模块（20项需求）

| 分类 | 模块 |
|------|------|
| 数据接入 | 企业微信审批同步、模板字段映射配置 |
| 基础数据 | 多账户管理（3公户+2私户）、客户与供应商管理 |
| 核心业务 | 合同管理、收付款管理、发票管理、报销与借款、专利库存管理 |
| 分析决策 | 交易项目管理（单笔利润）、成本管理、Dashboard看板 |
| 提醒通知 | 年费到期提醒（企业微信推送） |
| 对账核验 | 银行流水对账 |
| 工具 | 数据导出、历史数据迁移、自动识别归类 |
| 系统基础 | 角色权限、系统切换导航、企业微信用户匹配 |

## 关键设计决策

1. 审批同步后数据为"待确认"状态，财务需确认补充关联
2. 收付款区分"业务类"（关联合同）和"费用类"（关联成本类别）
3. 利润分"毛利润"（交易层面）和"净利润"（公司层面）两个口径
4. 专利入库为手动确认操作（付款≠权利转移）
5. 账户设置"期初余额"解决上线时数据不完整问题
6. 自动归类初期从简，纯关键词匹配

## 技术栈

- 前端：Vue 3 + Element Plus + Vite + ECharts
- 后端：Node.js + Express + Sequelize + MySQL + Redis
- 部署：腾讯云，子域名 `erp.iptt.top`，端口 3001
- 数据库：独立 `erp_db`，跨库访问主项目 users/patents 表
