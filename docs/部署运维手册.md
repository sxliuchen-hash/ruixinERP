## 三、部署流程

### 一键部署（推荐）

服务器上执行：

```bash
bash /var/www/erp/backend/scripts/deploy.sh
```

脚本会自动完成：
1. 拉最新代码（`git pull`）
2. 安装后端依赖
3. 构建前端（`npm run build`）
4. **拷贝前端产物到 Nginx 目录**（关键步骤）
5. 重启后端 PM2 服务

### 手动分步部署

```bash
# 1. 拉代码
cd /var/www/erp && git pull

# 2. 安装后端依赖（如有新依赖）
cd /var/www/erp/backend && npm install --production

# 3. 数据库迁移（如本次提交带 SQL 脚本）
mysql -u erp_db -p erp_db < /var/www/erp/backend/scripts/<迁移脚本>.sql

# 4. 构建前端
cd /var/www/erp/frontend && npm install && npm run build

# 5. 部署前端到 Nginx 目录（不能跳过！）
rm -rf /www/wwwroot/erp.iptt.top/*
cp -r /var/www/erp/frontend/dist/* /www/wwwroot/erp.iptt.top/

# 6. 重启后端
pm2 restart erp-backend

# 7. 浏览器强刷（Ctrl+Shift+R）
```

## 七、常见故障排查

### 问题 1：前端代码改了但页面没变化

**原因**：前端构建后没拷贝到 Nginx 服务目录。

**解决**：
```bash
ls -la /www/wwwroot/erp.iptt.top/index.html  # 确认时间是不是最新的
cp -r /var/www/erp/frontend/dist/* /www/wwwroot/erp.iptt.top/
# 浏览器 Ctrl+Shift+R 强刷
```

### 问题 2：服务启动失败

```bash
pm2 logs erp-backend --err --lines 50  # 看错误日志
tail -100 /var/www/erp/backend/logs/error.log
```

常见原因：
- 数据库连接失败：检查 `.env` 中 `DB_PASSWORD` 等
- 端口被占用：`lsof -i:3001`
- 缺少新依赖：`cd /var/www/erp/backend && npm install`

### 问题 3：定时任务没运行

确认日志里有注册信息：
```bash
grep "已注册" /var/www/erp/backend/logs/app.log | tail -10
```

### 问题 4：express-rate-limit 报 X-Forwarded-For 警告

已修复（`app.set('trust proxy', 1)`）。如再出现，确认 `backend/src/app.js` 中有该配置。

### 问题 5：调用 IP 系统 401

ERP 与 IP 系统共用 `JWT_SECRET`。如失败，确认 `.env` 中 `JWT_SECRET` 与 IP 系统的一致。

定时任务调用 IP 系统时，使用系统 Token（在 `patentBatchQueryJob.js` 中通过 `jwt.sign` 生成）。可在 `.env` 中配置：

```bash
IP_SYSTEM_USER_ID=1
IP_SYSTEM_USERNAME=system
```

## 十、重要提示

1. **永远不要** 直接编辑 `/www/wwwroot/erp.iptt.top` 下的文件，所有改动来自 git pull + 构建
2. **永远不要** 把 `.env` 提交到 git
3. **永远不要** 在生产数据库直接执行 `DROP TABLE / DELETE FROM` 等危险语句，先备份
4. **永远不要** 跳过 "拷贝到 Nginx 目录" 这一步
5. 部署后用浏览器无痕窗口验证（避开缓存干扰）
