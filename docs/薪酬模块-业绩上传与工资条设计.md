## 一、总体变更

人力薪资模块从"系统自动算业绩"调整为"**每月上传业绩统计表 → 核对确认 → 入库 → 算提成**"。

- 销售提成：来自每月人工上传的业绩表（核定业绩已在表中填好）
- 采购提成：系统从专利库存自动计算（仅采购人员卖出自营专利）
- 个税：简化版（按月独立计税），专项附加扣除字段预留
- 考勤扣款：事假/病假按法定标准
- 敏感页面（工资条、业绩看板、薪资规则、员工档案）叠加防泄露水印

## 二、销售业绩上传

### 2.1 流程

```
下载模板 → 填写上传 → 系统解析预览 → 人工核对确认 → 入库 → 自动算提成
```

每月上传一次，未确认前为草稿（可重传覆盖），确认后锁定参与提成计算。

### 2.2 上传表字段

| 列 | 说明 | 用途 |
|----|------|------|
| 姓名 | 业绩归属人 | 匹配员工（匹配不上预览标红，禁止确认） |
| 业务类型 | 发明买卖/实用买卖/软著/商标等 | 分类统计 |
| 流水号 | 业务流水号 | 追溯 |
| 专利号/项目号 | 标的编号 | 追溯 |
| 专利名/项目名 | 标的名称 | 追溯 |
| 合同金额 | 合同总额 | 留存 |
| 核定业绩 | **提成计算基数** | 按人按月合计后走超额累进 |
| 合同日期 | 签订日期 | 留存 |
| 尾款日期 | 回款完成日 | **决定提成发放月（尾款月+1）** |
| 是否全风险代理 | 是/否 | 仅记录，不影响核定业绩 |
| 备注 | | |

### 2.3 提成计算（超额累进）

每人当月「核定业绩」合计 T，套阶梯：

| 月度计提业绩 T | 提成率 |
|---------------|--------|
| 0–15000 元部分 | 10% |
| 15000–30000 元部分 | 15% |
| 30000–50000 元部分 | 18% |
| 50000 元以上部分 | 22% |

阶梯可在「薪资规则」页编辑（rule_type=commission，已有）。

### 2.4 提成发放时点

提成按**回款（尾款）完成的次月**随工资发放。即一条业绩归属「尾款日期所在月」，
提成体现在「尾款月 + 1」的工资条上。

## 三、采购提成（系统自动）

### 3.1 计算口径

- 仅采购人员（员工档案 role=purchase）卖出**自营**专利（库存 resource_type='own'）才计提
- 保留"卖出时间衰减"档位（rule_type=purchase_commission，已有）

| 卖出时间（卖出日−采购日） | 发明专利 | 实用新型 |
|--------------------------|---------|---------|
| 0–90 天 | 200 元/件 | 50 元/件 |
| 91–180 天 | 100 元/件 | 25 元/件 |
| 181–365 天 | 50 元/件 | 12.5 元/件 |
| 366–730 天 | 25 元/件 | 6.25 元/件 |
| >730 天 | 0 | 0 |

### 3.2 数据来源

- 库存记录 `resource_type='own'` 且 `status='sold'`
- 采购日 `purchase_date`、卖出日 `sold_time/stock_out_date`、类型 `patent_type`
- **采购人**：库存表新增字段 `purchaser_id`（关联员工/用户），提成归属到他
- 历史数据 `purchaser_id` 为空时，该条采购提成跳过

## 四、个税（简化版）

```
应纳税所得 = 应发工资 − 个人社保 − 起征点(5000)
按月独立计税，套月度税率表（非累计预扣）
专项附加扣除字段预留，初期填 0
```

- 参数可编辑（rule_type=income_tax）
- 后续可升级为累计预扣法 + 专项附加扣除录入

---

## 五、考勤扣款

- 日薪折算基数 = (基本工资 + 岗位补贴) ÷ 21.75（法定月计薪天数）
- 21.75 = (365 − 104 双休) ÷ 12，法定口径（劳社部发〔2008〕3号）
- **事假**：不带薪，`日薪 × 事假天数` 全扣
- **病假**：带薪打折，`日薪 × (1 − 病假系数) × 病假天数`；实发不低于最低工资 80%
- **全勤奖 100**：当月无任何请假即认定全勤；有请假即清零
- 年假/法定假/调休：不扣
- 参数可编辑（rule_type=attendance）

## 六、合伙人/老板

- **不生成工资条**，但单独报表能算出其收入
- 合伙人（罗正武）收入 = 本人业务毛利 × 70%，剩余 30% 归老板（刘晨）
- 公司业务统计范围排除刘晨、罗正武

---

## 七、工资条作废与调整

- 状态：draft（草稿）→ confirmed（已确认）→ paid（已发放），新增 **voided（已作废）**
- confirmed/paid 不可直接编辑，但可**作废**
- 更正方式：作废重开，或在下月工资条增加"调整项"（补发/补扣）

## 八、防泄露水印（已实现）

- 组件：`frontend/src/components/common/Watermark.vue`
- 实现：Canvas 平铺生成水印背景，pointer-events:none 不影响交互
- 文案：`姓名@账号` + `尾号XXXX 日期`（手机尾号取 4 位）
- 防篡改：MutationObserver 监听水印 DOM 删除/样式篡改，自动恢复
- 生效页面：通过路由 `meta.watermark=true` 控制
  - 工资条 `/payroll`
  - 业绩统计 `/performance`
  - 薪资规则 `/system/salary-rules`
  - 员工档案 `/employees`

## 九、数据表设计

### 9.1 performance_imports（业绩上传批次）

```sql
CREATE TABLE IF NOT EXISTS `performance_imports` (
  `id` INT NOT NULL AUTO_INCREMENT,
  `year` INT NOT NULL,
  `month` INT NOT NULL,
  `file_name` VARCHAR(255),
  `record_count` INT DEFAULT 0,
  `total_performance` DECIMAL(14,2) DEFAULT 0 COMMENT '核定业绩合计',
  `status` ENUM('draft','confirmed') DEFAULT 'draft',
  `uploaded_by` INT,
  `confirmed_by` INT,
  `confirmed_at` DATETIME,
  `create_time` DATETIME,
  `update_time` DATETIME,
  PRIMARY KEY (`id`),
  KEY `idx_year_month` (`year`,`month`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='业绩上传批次';
```

### 9.2 performance_records（业绩明细）

```sql
CREATE TABLE IF NOT EXISTS `performance_records` (
  `id` INT NOT NULL AUTO_INCREMENT,
  `batch_id` INT NOT NULL,
  `year` INT NOT NULL,
  `month` INT NOT NULL COMMENT '归属月（尾款日期所在月）',
  `employee_id` INT,
  `user_id` INT,
  `employee_name` VARCHAR(50) NOT NULL COMMENT '上传时填写的姓名',
  `business_type` VARCHAR(50) COMMENT '业务类型',
  `serial_no` VARCHAR(100) COMMENT '流水号',
  `target_no` VARCHAR(100) COMMENT '专利号/项目号',
  `target_name` VARCHAR(500) COMMENT '专利名/项目名',
  `contract_amount` DECIMAL(14,2) DEFAULT 0 COMMENT '合同金额',
  `performance_amount` DECIMAL(14,2) DEFAULT 0 COMMENT '核定业绩',
  `contract_date` DATEONLY,
  `final_payment_date` DATEONLY COMMENT '尾款日期',
  `is_full_risk_agent` BOOLEAN DEFAULT FALSE COMMENT '是否全风险代理（仅记录）',
  `remark` TEXT,
  `create_time` DATETIME,
  `update_time` DATETIME,
  PRIMARY KEY (`id`),
  KEY `idx_batch` (`batch_id`),
  KEY `idx_employee_month` (`employee_id`,`year`,`month`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='业绩明细';
```
