| 姓名 | 关联主项目 users.real_name |

## 二、业绩归属与统计

### 2.1 归属规则

- **业绩归属人 = 合同审批申请人（企微 applyer.userid）**
- 同步时已通过 `_resolveUserId(applyer.userid)` 写入 `contracts.created_by`
- 统计时按 `contracts.created_by` 聚合

### 2.2 销售毛利计算

- 个人月毛利 = 该人当月所有**已完成销售合同**的 `amount` 之和
- 注意：这里的"毛利"是合同金额（销售收入），不扣除采购成本
- 如果需要扣除对应采购成本，需通过 project 关联（后续可扩展）

### 2.3 统计维度

| 维度 | 用途 |
|------|------|
| 按人按月 | 计算月提成 |
| 按人按季度 | 判定职级 |
| 按人按年 | 年终考核 |

## 三、销售提成规则

### 3.1 月度提成（超额累进）

| 月毛利总额 T | 提成率 | 计算方式 |
|-------------|--------|---------|
| T ≤ 15,000 | 10% | 该区间部分 × 10% |
| 15,000 < T ≤ 30,000 | 15% | 超出 15000 的部分 × 15% |
| 30,000 < T ≤ 50,000 | 18% | 超出 30000 的部分 × 18% |
| T > 50,000 | 22% | 超出 50000 的部分 × 22% |

**超额累进计算示例**：月毛利 4 万
- 第一档：15000 × 10% = 1500
- 第二档：15000 × 15% = 2250
- 第三档：10000 × 18% = 1800
- 合计提成 = 1500 + 2250 + 1800 = **5550 元**

### 3.2 职级津贴（季度考核）

| 职级 | 季度累计毛利（万） | 月津贴 |
|------|-------------------|--------|
| A（初级） | < 6 | 0 |
| B（中级） | [6, 9) | 200 |
| C（高级） | [9, 12) | 500 |
| D（资深） | [12, 15) | 900 |
| E（专家） | ≥ 15 | 1400 |

- 考核时间：每年 3月、6月、9月、12月
- 考核期：上一个完整季度（如 3 月考核 1-3 月数据）
- 职级变更后下个月生效
- **达到目标即通提**（达到某档即全额享受该档津贴）

### 3.3 淘汰机制

| 条件 | 处理 |
|------|------|
| 试用期：连续 3 个月累计 < 6 万 | 转岗/延期/终止（三选一） |
| 转正后：连续 3 个月毛利 < 2 万/月 | 淘汰 |
| 连续 2 个季度为 A 级 | 视情况淘汰 |

## 四、采购提成规则

### 4.1 计算逻辑

- 采购人员负责采购专利（patent_inventory）
- 当采购的专利被卖出时（关联到销售合同），按**卖出时间距采购时间**计算提成
- 卖出天数 = 销售合同 sign_date - patent_inventory.purchase_date

### 4.2 提成阶梯

| 卖出时间 | 发明专利 | 实用新型 |
|---------|---------|---------|
| 0-3 个月（≤90天） | 200 元/件 | 50 元/件 |
| 3-6 个月（91-180天） | 100 元/件 | 25 元/件 |
| 6-12 个月（181-365天） | 50 元/件 | 12.5 元/件 |
| 12-24 个月（366-730天） | 25 元/件 | 6.25 元/件 |
| 24 个月以上（>730天） | 0 | 0 |

### 4.3 数据来源

- 采购日期：`patent_inventory.purchase_date`
- 专利类型：`patent_inventory.patent_type`（发明/实用新型）
- 卖出日期：关联的销售合同 `contracts.sign_date`（通过 project 或 contract 关联）
- 采购人：`patent_inventory.created_by`

## 五、合伙人收入

### 5.1 计算规则

- 合伙人（罗正武）的收入 = **其本人业务毛利 × 70%**
- 剩余 30% 归老板（刘晨）收入
- 这是挂靠业务模式，不是公司利润分红

### 5.2 老板（刘晨）收入

- 刘晨本人申请的业务收入：单独统计，不计入公司业务
- 合伙人业务的 30%：归入刘晨收入

### 5.3 公司业务统计范围

- **公司业务** = 除刘晨、罗正武之外的所有人的业务
- 刘晨和罗正武的业务**单独做统计报表**，不纳入公司整体业绩
- 公司总利润计算时不包含这两人的收入

### 5.4 年度分红

- 公司分红按年度整体利润计算（年终）
- 分红规则后续确定

### 5.5 报表展示

- 公司业务报表：4 个销售 + 1 个采购 + 1 个内勤
- 老板/合伙人报表：刘晨业务 + 罗正武业务（含 70%/30% 分成明细）

## 六、内勤薪资

### 6.1 计算规则

- 月薪 = 基本工资 + 岗位补贴 + 全勤 + 奖金
- 奖金：固定金额（admin 在系统中手动设置）
- 不参与提成计算
- 不参与职级考核

## 七、工资条

### 7.1 工资条结构

| 项目 | 说明 |
|------|------|
| 基本工资 | 从员工档案读取 |
| 岗位补贴 | 从员工档案读取 |
| 全勤奖 | 100 元，当月有请假则为 0（有请假就没全勤） |
| 职级津贴 | 根据当前职级自动填入 |
| 业务提成 | 销售：超额累进计算；采购：按专利卖出计算 |
| 奖金 | 内勤/特殊奖励（手动填写） |
| 社保扣除 | 按西安最低工资基数计算（个人部分） |
| 请假扣款 | 按法定标准：日薪 = 月薪 ÷ 21.75，按请假天数扣除 |
| 其他扣款 | 手动填写 |
| **实发合计** | 以上各项之和 |

### 7.1.1 社保公积金

- 基数：西安最低工资标准（系统可配置，当前约 2120 元）
- 个人缴纳比例（约）：养老 8% + 医疗 2% + 失业 0.3% + 公积金 5% = 约 15.3%
- 个人月扣除 ≈ 基数 × 15.3%（系统自动计算，基数和比例可在规则配置中修改）

### 7.1.2 请假规则

- 有请假 → 全勤奖为 0
- 请假扣款 = 请假天数 × 日薪
- 日薪 = (基本工资 + 岗位补贴) ÷ 21.75
- 法定节假日、年假等不扣款

### 7.2 生成流程

1. admin 点击"生成本月工资条"
2. 系统自动计算：基础薪资 + 职级津贴 + 提成
3. admin 手动补充：奖金、扣款
4. 确认后锁定，可导出 Excel

### 7.3 状态

- draft（草稿，可编辑）
- confirmed（已确认，不可改）
- paid（已发放）

## 八、主营业务成本分类

### 8.1 成本结构

```
主营业务成本
├── 专利年费（自动从 patent_annual_fees 汇总）
├── 专利采购（自动从采购合同汇总）
├── 供应商采购（按供应商分类）
├── 软著材料
│   ├── 普通材料：10 元/件
│   └── 高级材料：50 元/件（含 Token 消耗）
└── 其他直接成本

期间费用
├── 房租水电
├── 办公用品
├── 软件服务
├── 差旅招待
└── 其他

人力成本
├── 工资（从工资条汇总）
├── 社保公积金
└── 其他人力支出
```

### 8.2 软著材料成本

| 类型 | 单价 | 说明 |
|------|------|------|
| 普通材料 | 10 元/件 | 标准软著材料 |
| 高级材料 | 50 元/件 | 含 Token 消耗、AI 生成等 |

- 按月录入数量，系统自动计算金额
- 纳入主营业务成本

### 8.3 数据来源

| 成本项 | 数据来源 | 自动/手动 |
|--------|---------|----------|
| 专利年费 | patent_annual_fees 表 | 自动汇总 |
| 专利采购 | contracts(type=purchase) | 自动汇总 |
| 供应商采购 | payments(type=expense, category=business) | 自动汇总 |
| 软著材料 | 新增录入表 | 手动录入 |
| 房租水电等 | cost_records 表 | 已有（手动/固定月费） |
| 工资 | payroll 表 | 自动汇总 |

## 九、利润分析报表

### 9.1 计算公式

```
毛利润 = 销售收入 - 主营业务成本
净利润 = 毛利润 - 期间费用 - 人力成本
毛利率 = 毛利润 / 销售收入 × 100%
净利率 = 净利润 / 销售收入 × 100%
```

### 9.2 报表维度

- 按月/按季/按年
- 按项目
- 按客户
- 按人（销售业绩排名）
