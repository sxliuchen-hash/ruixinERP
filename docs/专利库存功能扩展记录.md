# 专利库存功能扩展记录

记录 2026-05 期间在专利库存模块增加的全部功能，便于后续维护时快速定位。

## 一、IP 系统年费数据对接

### 后端

| 文件 | 说明 |
|------|------|
| `backend/src/services/ipSystemService.js` | IP 系统 API 客户端，使用当前用户 JWT 调用 `https://iptt.top/api/v1` |
| `backend/src/controllers/patentFeeController.js` | 年费查询 Controller（薄层代理） |
| `backend/src/routes/patentFee.js` | 路由前缀 `/api/v1/patent-fee` |

提供的接口：
- `GET /api/v1/patent-fee/list` 年费列表
- `GET /api/v1/patent-fee/dashboard` 统计面板
- `GET /api/v1/patent-fee/detail/:patentNo` 单个专利完整年费详情
- `GET /api/v1/patent-fee/info/:patentNo` 专利基本信息

### 前端

- 详情页底部「国知局年费信息」卡片：展示应缴/已缴年费、法律状态、发文记录、变更记录
- 详情页顶部增加按钮：
  - **更新年费** — 重新拉取 IP 系统年费数据
  - **同步专利信息** — 把 IP 系统的专利名称/类型/截止日同步回 ERP 库存表

## 二、批量入库

### 后端

| 文件 | 说明 |
|------|------|
| `backend/src/services/inventoryBatchService.js` | Excel 模板生成、解析、批量入库（带 IP 系统补全） |

接口：
- `GET /api/v1/inventory/batch-import/template` 下载 Excel 模板
- `POST /api/v1/inventory/batch-import/validate` 上传校验（multer）
- `POST /api/v1/inventory/batch-import/execute` 确认入库

模板字段：专利号、采购价、定价、供应商名称、采购日期、是否报过高企、备注。其余信息（专利名称、类型）通过 IP 系统接口自动补全。

### 前端

- `frontend/src/views/inventory/BatchImportDialog.vue` 三步式弹窗（上传 → 预览 → 结果）
- 列表页「入库」按钮改为下拉菜单：单个入库 / 批量入库（Excel）

## 四、资源分类（自有/独家代理/共同代理）

### 数据库变更

`patent_inventory` 增加字段：
- `resource_type` ENUM('own','exclusive_agent','joint_agent') 默认 'own'
- `agent_id` INT（关联 suppliers.id）
- `profit_rule` JSON

迁移脚本：`backend/scripts/add-resource-type-and-anomaly.sql`

### 利润分成规则（profit_rule JSON 结构）

```json
{ "mode": "percent", "agent_share_pct": 30 }   // 代理商分 30%
{ "mode": "fixed", "agent_fixed_fee": 5000 }   // 代理商固定收 5000
```

`inventoryService._calculateNetProfit()` 自动计算：
- `own`：`net_profit = grossProfit`
- `exclusive_agent / joint_agent`：根据 rule 扣除分成

### 前端

- 列表筛选区新增「资源类型」下拉
- 表格增加「资源类型」列，下方显示代理商名称
- 入库/编辑表单：选择代理类型时显示代理商选择器 + 分成规则配置（按百分比 / 固定费用）
- 详情页基本信息显示资源类型 + 代理商
- 详情页财务摘要：自有显示「利润预估」；代理类型同时显示「毛利润」「实得利润（扣分成）」

## 五、专利异常告警

### 数据库

新建表 `patent_anomaly_alerts`：
- `anomaly_type`：pledge / license / change / transfer_fee / legal_status / other
- `severity`：info / warning / danger
- `dedupe_key`：去重键，避免重复告警

### 后端

| 文件 | 说明 |
|------|------|
| `backend/src/models/PatentAnomalyAlert.js` | 模型 |
| `backend/src/services/patentAnomalyService.js` | 异常检测 + 告警入库 + 通知推送 |
| `backend/src/jobs/patentBatchQueryJob.js` | 周日 06:00 定时扫描 |

检测规则（30 天内出现）：
| 类型 | 触发条件 | 严重程度 |
|------|---------|---------|
| pledge | 新增质押 | warning |
| license | 新增许可 | warning |
| change | 新增著录变更 | **danger（疑似被转走）** |
| transfer_fee | 出现"著录项目变更费" | **danger（疑似已转让）** |
| legal_status | 法律状态变为终止/无效宣告/权利转移 | **danger** |

### 接口

- `GET /api/v1/inventory/anomalies` 告警列表（分页、筛选）
- `GET /api/v1/inventory/anomalies/overview` 统计
- `PUT /api/v1/inventory/anomalies/:id/resolve` 标记处理
- `POST /api/v1/inventory/anomalies/scan` 手动触发扫描（仅管理员）

### 系统级 Token

定时任务无 HTTP 上下文，通过 `jwt.sign` 直接签发短期 Token 调用 IP 系统。可在 `.env` 中配置：

```bash
IP_SYSTEM_USER_ID=1
IP_SYSTEM_USERNAME=system
```

未配置时使用默认 `id=1, username=system, role=admin`。

### 前端

- `frontend/src/views/inventory/PatentAnomalyAlerts.vue` 异常告警中心页面
- 路由：`/inventory/anomalies`
- 4 个统计卡片，danger 卡片有红色脉冲动画
- 列表行用红/橙色高亮严重程度
- 列表页顶部「异常告警」按钮带 badge（danger 数量）
- 侧边栏「专利库存」改为子菜单：库存列表 / 异常告警

## 九、注意事项

1. **批量入库** 会调用 IP 系统接口为每个专利补全名称/类型，建议单次不超过 100 条（IP 系统限流 800ms/请求）
2. **周扫描** 在周日 06:00 自动运行，全量扫描在库专利。如紧急排查可点「立即扫描」按钮
3. **资源类型** 新增专利默认为 `own`（自有），代理类型必须填写代理商和分成规则
4. **批量删除** 仅 admin 可见和可调用，service 层有双重校验（前端隐藏 + 后端 requireAdmin 中间件）
5. **告警去重** 通过 `dedupe_key` 实现，相同事件不会重复告警
